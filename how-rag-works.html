Retrieval-Augmented Generation

How RAG Works

A way to give language models an open-book exam — letting them look up trusted, up-to-date facts before they answer, instead of relying on memory alone.

Why RAG exists

A large language model only knows what it saw during training. That creates real limits — RAG is designed to fix them.

● A plain LLM alone

  • Knowledge is frozen at its training cutoff
  • Can't see your private or internal documents
  • May hallucinate confident but wrong answers
  • Hard to trace where an answer came from

● An LLM with RAG

  • Pulls in fresh, current information on demand
  • Grounds answers in your own knowledge base
  • Fewer hallucinations — answers cite real sources
  • Update knowledge by updating documents, not retraining
📚

The open-book exam analogy

Imagine a brilliant student taking a test. On a closed-book exam, they answer from memory — sometimes fuzzy, sometimes wrong. RAG hands them the open textbook: they quickly find the relevant pages, read them, and then write an answer grounded in what the book actually says. The intelligence is the same — but now it's backed by evidence.

Before any of that: indexing

RAG has a behind-the-scenes prep phase. Your knowledge base is processed once (and refreshed as it changes) so it's ready to search instantly.

Step A

Chunk

Long documents are split into bite-sized passages — small enough to be precise, large enough to keep context.

Step B

Embed

Each chunk is converted into a vector by an embedding model, so similar meanings sit near each other in "concept space."

Step C

Store

Vectors go into a vector database (like Pinecone, Weaviate, or FAISS) built for lightning-fast similarity search.

What "similarity" actually means

Embeddings place text in a high-dimensional space where distance equals meaning. Retrieval simply grabs the nearest neighbors to your question.

Your question 💬 [0.82, 0.14, 0.55, …]
"Return policy" doc
94% match
"Shipping times" doc
61% match
"Office holidays" doc
18% match

Key idea: RAG doesn't match keywords — it matches meaning. A question about "getting my money back" can retrieve a "refunds" document even if those exact words never appear.

Putting it together in the prompt

The magic of "augmentation" is simple: retrieved context is stitched into the prompt the model receives.

System: Answer using only the context below.
Context: "Items may be returned within 30 days for a full refund…"
User question: How long do I have to return something?
Model: You have 30 days from purchase to return an item for a full refund.
